# 音乐播放器项目文档集合

## 项目文档：Audio Flow (C#)

---

## 1. 项目概述

**Audio Flow** 是一款基于 .NET 9 的高性能、跨平台频谱可视化音乐播放器。本项目旨在为追求极致视听体验的开发者及音乐发烧友提供工业级的音频处理和可视化方案，同时压榨 C# 在多媒体处理领域的底层性能。

### 1.1 项目目标

- 实现低延迟、零内存分配的音频捕获与处理管道
- 提供多维度、可定制的实时频谱可视化方案
- 支持本地文件播放与系统音频全局捕获
- 跨平台兼容（Windows、macOS、Linux）
- 提供插件化的可视化扩展机制

### 1.2 核心技术栈

| 组件 | 技术方案 | 说明 |
|------|--------|------|
| **Runtime** | .NET 9 LTS | 跨平台运行时，性能优化 |
| **UI Framework** | Avalonia UI | MVVM 模式，跨平台原生渲染 |
| **音频处理** | NAudio (Windows) / ManagedBass (跨平台) | WASAPI 环回、文件解码 |
| **图形渲染** | SkiaSharp + OpenGL | GPU 加速，高性能绘制 |
| **DSP 库** | System.Numerics (SIMD) | 向量化计算，频谱处理 |

## 2. 软件架构设计 (SDD)

### 2.1 架构设计原则

1. **分层设计**：清晰的模块边界，便于测试与扩展
2. **零拷贝数据流**：使用 `Span<T>` 和 `Memory<T>` 最小化内存分配
3. **线程安全**：音频线程与 UI 线程隔离，通过无锁数据结构通信
4. **可插拔化**：可视化、音频驱动、主题系统均支持插件扩展

### 2.6 安全性考虑

#### 2.6.1 音频权限与隔离

| 威胁 | 风险等级 | 缓解措施 |
|------|--------|--------|
| **WASAPI 环回权限不足** | 中 | 检测权限失败时降级到文件播放模式，提示用户 |
| **恶意音频文件（畸形 MP3/FLAC）** | 中 | 使用 NAudio 的异常捕获，隔离解码错误，防止应用崩溃 |
| **系统音频隐私泄露** | 低 | 仅在本地捕获音频，不上传任何数据；提供隐私政策 |
| **插件恶意代码执行** | 高 | 限制插件权限：禁止文件系统访问、网络访问；使用 AppDomain 沙箱隔离（.NET Framework）或程序集加载限制 (.NET Core) |

#### 2.6.2 数据安全

- **数据库加密**: SQLite 可选使用 SQLCipher 加密敏感配置
- **敏感信息存储**: 避免存储用户隐私数据（如播放习惯分析）；播放历史仅本地保存
- **文件访问控制**: 验证文件路径合法性，防止目录遍历攻击

#### 2.6.3 网络安全（预留）

- 若未来支持在线功能（如歌词同步、云同步），需实现 HTTPS + 身份验证
- API 调用使用 JWT Token，支持刷新机制

#### 2.6.4 代码安全

- **输入验证**: 所有用户输入（文件路径、配置值）需验证格式
- **异常处理**: 所有外部调用（文件 I/O、音频驱动）需 try-catch，防止未捕获异常导致应用崩溃
- **依赖库安全**: 定期更新 NuGet 包，使用 `dotnet list package --vulnerable` 检查已知漏洞

#### 2.6.5 用户隐私

- **默认不收集数据**: 应用完全离线运行，不向任何服务器发送用户数据
- **透明的权限请求**: 访问文件系统、音频设备时明确提示用户
- **数据本地化**: 所有配置、播放列表、历史记录均存储在用户本地目录，不同步到云端

### 2.7 性能目标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| **UI 帧率** | ≥60 FPS | 平滑的可视化效果 |
| **音频延迟** | <50ms | 从音频输入到频谱更新 |
| **内存占用** | <200MB | 基础运行状态 |
| **CPU 占用** | <15% | 单核占用率（i7 8700K 参考） |
| **启动时间** | <2s | 应用启动到可用 |
| **FFT 处理延迟** | <10ms | 8192 点 FFT 变换时间 |

### 2.8 扩展性设计

#### 2.8.1 插件系统

通过 MEF (Managed Extensibility Framework) 或自定义反射机制实现插件化：

- **可视化插件**: 实现 `IVisualizer` 接口，放入 `Plugins/Visualizers/` 目录
- **主题插件**: 实现 `ITheme` 接口，放入 `Plugins/Themes/` 目录
- **音频驱动插件**: 实现 `IAudioProvider` 接口，支持自定义音频源

#### 2.8.2 配置热加载

支持不重启应用即时加载新的可视化器、主题配置，通过 `FileSystemWatcher` 监听插件目录变化。

## 3. 模块详细设计

### 3.1 音频层 (Audio Layer)

**职责**: 从系统或文件中捕获原始 PCM 数据，提供统一的数据接口。

**关键设计**:
- 使用 `Span<float>` 避免数据拷贝
- `ArrayPool<float>` 管理临时缓冲区
- 高优先级音频线程处理实时数据
- 跨平台适配（Windows WASAPI / Linux PulseAudio / macOS CoreAudio）

### 3.2 DSP 层 (Digital Signal Processing Layer)

**职责**: 对 PCM 数据进行窗口函数处理、FFT 变换、频率加权、平滑处理。

**关键设计**:
- 支持可配置的窗口函数（Hann、Hamming、Blackman）
- 动态 FFT 窗口大小（512 ~ 8192）
- A-weighting 和对数缩放频率加权
- 重力下降（Gravity）和线性插值（Lerp）平滑算法

### 3.3 可视化层 (Visualization Layer)

**职责**: 将频率数据映射为可视化图形，支持多种渲染模式。

**关键设计**:
- 基于 SkiaSharp 的 GPU 加速渲染
- 支持渐变色、粒子效果、轨道动画
- 可配置的渲染参数（颜色、速度、大小）
- 插件化架构，支持自定义可视化器

### 3.4 UI 层 (User Interface Layer)

**职责**: 提供用户交互界面，管理主题、设置、播放列表。

**关键设计**:
- MVVM 模式，ViewModel 与 View 分离
- 响应式布局，支持窗口缩放
- 主题系统，支持深色/浅色切换
- 拖拽上传文件、快捷键支持

### 3.5 数据层 (Data Layer)

**职责**: 管理应用配置、播放列表、播放历史的持久化存储。

**关键设计**:
- Entity Framework Core ORM 映射
- Repository 模式隔离数据访问
- 支持数据库迁移和版本管理
- 异步数据操作，避免阻塞 UI

### 3.6 核心层 (Core Layer)

**职责**: 提供日志、事件、配置等基础服务。

**关键设计**:
- 中央事件总线，解耦模块通信
- 结构化日志（Serilog）
- 配置管理（支持 JSON、YAML）
- 依赖注入容器（Microsoft.Extensions.DependencyInjection）

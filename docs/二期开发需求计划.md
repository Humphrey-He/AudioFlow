## 一、需求概述

### 1.1 背景

一期已完成核心功能：
- 实时音频频谱可视化
- WebSocket 实时数据流
- Canvas 高性能渲染
- 多语言支持 (EN/ZH/JA)
- 基础视觉效果 (Glow/Reflection/Peak/Pulse/Line)

### 1.2 二期目标

在稳定的一期基础上，扩展可视化模式、增强音频分析能力、提升用户体验。

### 1.3 需求分类

| 分类 | 需求数量 | 优先级 |
|------|---------|--------|
| 可视化模式 | 6 | P1-P3 |
| 音频源扩展 | 4 | P1-P2 |
| 分析功能 | 5 | P2 |
| 社交/娱乐 | 4 | P2-P3 |
| 专业工具 | 4 | P3 |
| PWA/移动端 | 3 | P2 |
| 主题定制 | 3 | P3 |

## 二、功能需求详细说明

### 2.1 可视化模式扩展

#### 2.1.1 瀑布图模式 [P1]

**功能描述**
显示频谱随时间变化的历史轨迹，形成类似瀑布的三维效果。

**用户场景**
- 观察音乐频率变化趋势
- 分析声音衰减特性
- 长时间声音模式识别

**技术方案**
```
实现方式:
- 使用双缓冲 Canvas 或 OffscreenCanvas
- 保留最近 N 帧 (如 100 帧) 的频谱数据
- 每帧数据向下滚动一格
- 使用色温图 (cool to warm) 着色

性能目标:
- 60fps 渲染
- 内存占用 < 50MB
- 支持 100+ 帧历史
```

**UI/UX 设计**
- 新增模式切换 Tab: [频谱] [瀑布] [3D]
- 瀑布高度/宽度可调节
- 颜色主题可切换 (火焰/极光/科技)

**验收标准**
- [ ] 切换到瀑布模式后正确显示
- [ ] 滚动流畅无卡顿
- [ ] 支持拖动调节历史帧数

#### 2.1.3 环形/极坐标频谱 [P1]

**功能描述**
将线性频谱转换为从中心向外辐射的圆形/半圆形显示。

**用户场景**
- 更符合人耳对频率的感知习惯
- 视觉效果独特
- 适合电子音乐

**技术方案**
```
实现方式:
- 将频率映射到角度 (低频在中心，高频在外围)
- 条形从中心向外延伸
- 支持 360° 全圆或 180° 半圆

数学映射:
- 角度 = (freqIndex / totalBins) * (maxAngle - minAngle)
- 半径 = baseRadius + magnitude * scaleFactor
```

**UI/UX 设计**
- 圆心显示整体能量
- 顺时针/逆时针可选
- 低频区域高亮显示

**验收标准**
- [ ] 圆形频谱正确渲染
- [ ] 视觉效果符合预期
- [ ] 可切换全圆/半圆

#### 2.1.6 乐器指示 [P3]

**功能描述**
根据频谱峰值自动识别并标注对应乐器。

**技术方案**
```
频率-乐器映射:
- 60-250Hz: 贝斯
- 250-500Hz: 低音吉他
- 500-2000Hz: 吉他/钢琴/人声
- 2000-4000Hz: 人声中高频
- 4000-8000Hz: 镲片/高频泛音
- 8000-20000Hz: 极高频

识别算法:
- 找到前 3 个峰值频率
- 匹配到对应乐器
- 在频谱上显示标签
```

**验收标准**
- [ ] 主要乐器频率正确标注
- [ ] 支持同时显示多个标签

### 2.3 分析功能

#### 2.3.1 节拍检测 [P2]

**功能描述**
自动检测音频节拍 (BPM)，显示节拍标记。

**技术方案**
```
检测算法:
1. 计算频谱能量包络
2. 使用峰值检测找到能量突变点
3. 计算相邻峰值间隔
4. 平均得到 BPM

显示方式:
- 在频谱下方显示节拍线
- 显示当前 BPM 数值
- 可调节灵敏度
```

**验收标准**
- [ ] BPM 检测准确 (误差 < 5%)
- [ ] 节拍标记与音频同步
- [ ] BPM 数值实时更新

#### 2.5.2 测量模式 [P3]

**功能描述**
显示 SPL (声压级) 分贝值。

**技术方案**
```
计算公式:
SPL = 20 * log10(rms / ref)
ref = 0.00002 Pa (20µPa)

显示:
- 当前 SPL
- 峰值 SPL
- 平均 SPL (Leq)
```

**验收标准**
- [ ] SPL 计算准确
- [ ] 支持校准
- [ ] 显示 dB(A) / dB(C) / dB(Z)

## 五、风险评估

| 风险 | 影响 | 缓解措施 |

#### **RFC-005: 跨平台音频捕获与驱动适配**

**功能定位**
- 定义 IAudioProvider 接口的统一规范
- 规范 Windows WASAPI、Linux PulseAudio、macOS CoreAudio 的适配方案
- 明确音频缓冲区管理与内存池策略

**核心要求**
- 提供统一的 IAudioProvider 接口，屏蔽平台差异
- Windows：使用 NAudio WASAPI 环回，支持多设备选择
- Linux/macOS：使用 ManagedBass 跨平台库
- 采用 ArrayPool<float> 管理临时缓冲区，零分配设计
- 支持采样率自适应（44.1kHz / 48kHz / 96kHz）

**设计范围**
- IAudioProvider 接口详细设计
- 平台特定实现类（WasapiAudioProvider / PulseAudioProvider / CoreAudioProvider）
- 音频缓冲区设计（环形缓冲区、双缓冲）
- 采样率转换与重采样算法
- 设备枚举与选择机制
- 权限检查与降级策略

**交付物**
- 跨平台音频适配指南
- IAudioProvider 接口规范文档
- 各平台实现的参考代码
- 缓冲区管理与内存池示例
- 测试覆盖清单（单设备、多设备、权限不足场景）

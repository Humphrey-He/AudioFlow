## 简介

AudioFlow 是一款实时音频频谱分析器和可视化工具。它能够：
- 实时捕获系统音频或麦克风输入
- 执行 FFT 频谱分析
- 以可视化方式展示音频频率分布
- 支持多种平滑算法和频率加权

### 系统要求

| 要求 | 最低配置 | 推荐配置 |
|------|---------|---------|
| 操作系统 | Windows 10+ | Windows 11 |
| 内存 | 4GB | 8GB+ |
| 音频 | 支持系统音频捕获 | 专业声卡 |
| 网络 | 仅前端不需要 | 后端需要 localhost |

## 快速开始

### 方式一：使用后端服务器（推荐）

**步骤 1：启动后端服务**

```bash
cd src/AudioFlow.Server
dotnet run
```

后端服务将在 `http://localhost:5000` 启动。

**步骤 2：打开浏览器**

在浏览器中访问：`http://localhost:5000`

**步骤 3：授予音频权限**

浏览器会请求麦克风或系统音频访问权限，请选择允许。

### 方式二：仅使用前端（本地开发）

```bash
cd src/AudioFlow.Client
npm install
npm run dev
```

前端开发服务器将在 `http://localhost:3000` 启动。

## 功能详解

### 1. 连接状态

| 状态 | 颜色 | 说明 |
|------|------|------|
| Connected | 绿色 | 已连接到后端，正在接收数据 |
| Disconnected | 红色 | 未连接 |
| Connecting | 黄色闪烁 | 正在连接 |
| Reconnecting | 黄色闪烁 | 断线重连中 |
| Error | 红色 | 连接错误 |

### 2. 频谱可视化

- **横轴**：频率范围 (20Hz - 20kHz，对数刻度)
- **纵轴**：音量强度 (-60dB 到 0dB)
- **颜色渐变**：绿色(低频) → 紫色(中频) → 红色(高频)

### 3. 控制面板

#### 平滑算法 (Smoothing Algorithm)

| 选项 | 说明 |
|------|------|
| Gravity (默认) | 自然指数响应，对突发声音快速攻击，渐进衰减 |
| Linear Interpolation | 以恒定速率在值之间平滑过渡 |
| None | 原始数据无平滑，显示精确瞬时值 |

#### 攻击速率 (Attack Rate)

控制幅度向更高值上升的速度。

- **范围**：0.1 - 1.0
- **默认值**：0.6
- **效果**：值越高，对突发响亮声音响应越快

#### 衰减速率 (Decay Rate)

控制声音停止后幅度下降的速度。

- **范围**：0.1 - 1.0
- **默认值**：0.2
- **效果**：值越高，声音停止后衰减越快

#### 频率加权 (Frequency Weighting)

模拟人耳对不同频率的敏感度。

| 选项 | 说明 |
|------|------|
| A-Weighting (默认) | 遵循人耳频率响应曲线，强调中频 (2-5kHz) |
| C-Weighting | 更平坦的响应，适合高音量测量 |
| Linear | 无加权，显示实际测量值 |

### 4. 视觉效果

| 效果 | 说明 | 建议场景 |
|------|------|---------|
| Glow | 高能量条形发光 (> -20dB) | 通用 |
| Reflection | 条形下方显示镜像倒影 | 增加层次感 |
| Peak Hold | 峰值点保持 1.5 秒后衰减 | 观察最大电平 |
| Background Pulse | 背景随低频能量脉动 | 电子音乐 |
| Center Line | 条形中点连线 | 观察频谱形状 |

### 5. 效果预设

| 预设 | 包含效果 | 适用场景 |
|------|---------|---------|
| Default | Glow + Reflection + Peak | 日常使用 |
| Bassic | 全部开启 | 电子音乐、舞曲 |
| Vivid | 全部开启 | 展示、截图 |
| Minimal | 全部关闭 | 低功耗设备 |
| Custom | 用户自定义 | 个人偏好 |

### 6. 统计面板

| 指标 | 说明 | 正常范围 |
|------|------|---------|
| Frame | 当前帧计数器 | 持续增长 |
| Peak dB | 所有频率中的最大幅度 | -60dB ~ 0dB |
| Avg dB | 所有频率的平均幅度 | 反映整体响度 |
| FPS | 每秒渲染帧数 | 目标 60fps |

### 7. 诊断面板

显示技术信息用于故障排除：

| 指标 | 说明 |
|------|------|
| Messages | 收到的消息总数 |
| Invalid | 无效/被拒绝的消息数 |
| Reconnects | 重连次数 |
| Last Msg | 最后收到消息的时间 |
| Runtime Ver | 运行时版本号 |
| Last Error | 最后发生的错误 |

## 参数配置

### 推荐配置

| 场景 | 平滑 | Attack | Decay | 加权 |
|------|------|--------|-------|------|
| 通用监听 | Gravity | 0.6 | 0.2 | A |
| 语音聊天 | Gravity | 0.8 | 0.3 | A |
| 音乐欣赏 | Gravity | 0.4 | 0.1 | A |
| 噪音检测 | None | - | - | Linear |
| 专业测量 | Gravity | 0.3 | 0.05 | C |

### 效果组合建议

| 目标 | 推荐效果组合 |
|------|-------------|
| 最低延迟 | 无平滑 + Glow |
| 最美观的 | 全部开启 |
| 性能优先 | 仅 Glow |
| 观察峰值 | Peak Hold + Glow |
| 沉浸体验 | Pulse + Reflection |

## 视觉效果详解

### Glow (发光效果)

**触发条件**：单个频率幅度 > -20dB

**视觉效果**：
- 条形顶部出现白色高光
- 发光强度随幅度增加
- 模拟真实灯光效果

### Reflection (倒影)

**视觉效果**：
- 每个条形下方显示倒影
- 倒影高度为原条形的 30%
- 透明度 15%
- 增加视觉深度

### Peak Hold (峰值保持)

**行为**：
- 白色圆点标记当前峰值位置
- 保持 1.5 秒
- 然后 0.5 秒渐变衰减到当前值

### Background Pulse (背景脉动)

**触发**：低频能量变化 (20Hz-200Hz)

**视觉效果**：
- 背景明暗随低频节奏变化
- 透明度变化范围 0-15%
- 创造沉浸式音乐节奏感

### Center Line (中心线)

**视觉效果**：
- 连接所有条形中点的平滑曲线
- 紫色半透明
- 带发光效果
- 便于观察整体频谱形状

## 常见问题

### Q1: 页面显示"无法连接服务器"

**原因**：后端服务器未运行

**解决方法**：
```bash
# 启动后端
dotnet run --project src/AudioFlow.Server
```

### Q2: 连接成功但看不到频谱

**原因**：没有音频输入或麦克风权限被拒绝

**解决方法**：
1. 检查浏览器音频权限设置
2. 尝试播放音乐或说话
3. 检查系统音频输入设备

### Q3: FPS 很低或卡顿

**原因**：视觉效果过多或设备性能不足

**解决方法**：
1. 切换到 Minimal 预设
2. 关闭不必要的视觉效果
3. 减少打开的其他应用程序

### Q4: 统计数据不更新

**原因**：WebSocket 连接异常

**解决方法**：
1. 刷新页面
2. 检查后端日志
3. 查看诊断面板的错误信息

### Q5: 如何查看更多技术信息

**方法**：点击右上角的 `? 帮助` 按钮，可查看：
- 入门指南
- 频率范围说明
- 关于 AudioFlow
